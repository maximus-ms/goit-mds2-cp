# Тиждень 1. Щотижневий звіт про прогрес виконання дипломної роботи

## 1. Опис виконаних завдань та досягнень

На цьому тижні було проведено фундаментальну підготовку до активної фази розробки:

### Формування теми та концепції

В ході серії досліджень та консультацій було остаточно сформульовано тему дипломного проєкту. Основний акцент зміщено з простого аудіо-моніторингу на створення інтелектуальної системи, що використовує **мікрофонну решітку** для покращення якості вхідних даних у промислових умовах.

### Визначення методології ML

Прийнято стратегічне рішення відмовитися від класичної класифікації типів поломок (яка потребує величезних розмічених датасетів для кожного виду обладнання). Замість цього розробляється універсальна **модель-фіче-екстрактор**.

**Суть підходу:** Модель навчається виділяти ключові ознаки "здорового" сигналу. Будь-яке суттєве відхилення від еталонного вектора ознак розцінюється як зміна режиму роботи або нештатна ситуація. Це робить систему універсальною для різних типів агрегатів.

### Обґрунтування апаратної платформи

Підтверджено доцільність використання **ESP32-S3**. Завдяки наявності векторних інструкцій (S3-series), цей МК дозволить виконувати інференс розробленої моделі-екстрактора безпосередньо на пристрої (**Edge AI**).

## 2. Визначені ризики та виклики

### Виклик 1: Обчислювальна складність моделі на Edge-пристрої

**Проблема:** Складно передбачити, чи вистачить потужності ESP32-S3 для обраної архітектури нейромережі.

**Шлях вирішення:** На першому етапі тренування та аналіз моделі буде проводитися на ПК з використанням GPU. Це дозволить провести бенчмаркінг різних архітектур, знайти найлегшу ("наймолодшу") модель з достатньою точністю, а вже потім адаптувати її (квантування, прунінг) для розгортання на мікроконтролері.

### Виклик 2: Якість даних у шумному середовищі

**Вирішення:** Використання мікрофонної решітки замість одного мікрофона для реалізації просторової фільтрації сигналу перед подачею на вхід моделі.

## 3. План роботи на наступний тиждень

Наступний етап присвячений переходу від теорії до збору реальних даних та створення інфраструктури проєкту:

- **Розробка фізичного прототипу:** Складання пристрою на базі ESP32-S3 та мікрофонної решітки, що здатний здійснювати запис звуку.
- **Створення системи передачі даних:** Налаштування передачі аудіопотоку з пристрою на ПК для формування первинного датасету.
- **Первинний аналіз:** Проведення спектрального аналізу отриманих записів на ПК для валідації роботи заліза.
- **Відлагодження:** Усунення апаратних шумів та налаштування стабільної роботи DMA-буферів.
- **Інфраструктура:** Створення репозиторію на GitHub та документування початкового коду.
